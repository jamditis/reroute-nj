뉴스룸 및 출판사를 위한 안내

Reroute NJ는 기자, 뉴스룸, 커뮤니티 출판사를 위해 설계되었습니다.

기사에 임베드하기

  • 위의 설정 도구를 사용하여 어떤 형식이든 삽입 코드를 생성하세요.
  • CMS(WordPress, Ghost, Squarespace 등)에 붙여넣으세요 — 대부분 원시 HTML 블록을 지원합니다.
  • 삽입 콘텐츠는 반응형이며 모바일에서도 작동합니다.
  • API 키나 계정이 필요 없습니다.

기사에서 링크하기

  • Portal Bridge 기사에 "통근 계획하기" 링크 박스를 추가하세요.
  • 기사 내용에 맞는 특정 도구에 링크하세요.
  • OG 이미지와 소셜 카드가 깔끔한 미리보기를 위해 이미 설정되어 있습니다.
  • 추천 앵커 텍스트: "무료 Reroute NJ 도구로 통근을 계획하세요."

기사 제보

  • 뉴스 페이지에서 Portal Bridge 관련 보도를 모아두고 있습니다.
  • 기사를 제안하려면 GitHub에서 이슈를 열어주세요.
  • 또는 amditisj@montclair.edu로 "Reroute NJ coverage"를 제목에 넣어 이메일을 보내주세요.
  • 오피니언, 분석, 커뮤니티 보도 등 관련 기사를 모두 포함합니다.

공동 브랜딩 및 맞춤 설정

자체 브랜딩으로 Reroute NJ를 운영하고 싶으신가요? 오픈소스이므로 가능합니다.

프로젝트 포크

1

GitHub 저장소를 조직의 계정으로 포크하세요.

2

css/styles.css를 편집하여 색상, 폰트, 브랜딩을 변경하세요. 모든 테마 값은 :root 블록의 CSS 커스텀 속성입니다.

3

브랜드 이름, 로고, 푸터 크레딧을 자신의 것으로 교체하세요.

4

GitHub Pages, Netlify, Vercel 또는 자체 서버에 배포하세요. 빌드 단계가 필요 없습니다.

기여 방법

Reroute NJ는 커뮤니티 프로젝트입니다. 다음과 같이 도움을 줄 수 있습니다.

비기술적 기여

  • Submit article suggestions: Know of a Portal Bridge article we missed? Open an issue or email amditisj@montclair.edu.
  • 오류 신고: 시간표 데이터나 환승 소요 시간이 잘못된 경우 알려주세요.
  • 도구 공유: 통근하는 지인에게 알려주고, 소셜 미디어에 공유하고, 동네 커뮤니티에 올려주세요.

기술적 기여

  • 버그 신고: 버그를 발견하셨나요? GitHub에서 이슈를 등록하세요.
  • 풀 리퀘스트 제출: 코드베이스는 빌드 과정 없는 순수 HTML/CSS/JS입니다. 포크하고 수정하고 PR을 보내세요.
  • 데이터 추가: data/coverage.json에 새 기사를 업데이트하는 것을 도와주세요.
  • 접근성 개선: 스크린 리더 테스트, WCAG 준수, 키보드 내비게이션.
