Para sa mga newsroom at publisher

Ang Reroute NJ ay dinisenyo para sa mga mamamahayag, newsroom, at community publisher.

I-embed sa iyong mga artikulo

  • Gamitin ang configurator sa itaas para gumawa ng embed code sa anumang format.
  • I-paste sa iyong CMS (WordPress, Ghost, Squarespace, atbp.) — karamihan ay sumusuporta sa raw HTML blocks.
  • Ang mga embed ay responsive at gumagana sa mobile.
  • Hindi kailangan ng API key o account.

Mag-link mula sa iyong balita

  • Magdagdag ng "Planuhin ang iyong biyahe" na link box sa iyong mga artikulo tungkol sa Portal Bridge.
  • Mag-link sa mga partikular na tool batay sa nilalaman ng artikulo.
  • Ang mga OG image at social card ay naka-set up na para sa maayos na preview.
  • Inirerekomendang anchor text: "Gamitin ang libreng Reroute NJ tool para planuhin ang iyong biyahe."

Mag-ambag ng balita

  • Nagko-curate kami ng balita tungkol sa Portal Bridge sa aming pahina ng mga balita.
  • Para magmungkahi ng artikulo, mag-open ng issue sa GitHub.
  • O mag-email sa amditisj@montclair.edu na may "Reroute NJ coverage" sa subject.
  • Kasama namin ang lahat ng kaugnay na balita — opinyon, pagsusuri, community reporting, at iba pa.

Co-branding at customization

Gusto mo bang patakbuhin ang Reroute NJ gamit ang sarili mong branding? Dahil open source ito, kaya mo.

I-fork ang proyekto

1

I-fork ang GitHub repository sa account ng iyong organisasyon.

2

I-edit ang css/styles.css para baguhin ang kulay, font, at branding. Lahat ng theme value ay CSS custom property sa :root block.

3

Palitan ang brand name, logo, at footer credit ng iyong sarili.

4

I-deploy sa GitHub Pages, Netlify, Vercel, o sa sarili mong server. Hindi kailangan ng build step.

Paano mag-ambag

Ang Reroute NJ ay isang community project. Narito kung paano ka makakatulong.

Mga hindi teknikal na kontribusyon

  • Submit article suggestions: Know of a Portal Bridge article we missed? Open an issue or email amditisj@montclair.edu.
  • Mag-ulat ng pagkakamali: Kung mali ang datos ng iskedyul o oras ng transfer, ipaalam sa amin.
  • Ibahagi ang tool: Sabihin sa iyong mga kaibigang commuter, ibahagi sa social media, i-post sa iyong neighborhood group.

Mga teknikal na kontribusyon

  • Mag-ulat ng bug: Nakakita ng bug? Mag-bukas ng issue sa GitHub.
  • Mag-submit ng pull request: Ang codebase ay vanilla HTML/CSS/JS na walang build step. I-fork, baguhin, mag-PR.
  • Magdagdag ng datos: Tulungan kaming panatilihing updated ang data/coverage.json ng mga bagong artikulo.
  • Pagbutihin ang accessibility: Screen reader testing, WCAG compliance, keyboard navigation.
